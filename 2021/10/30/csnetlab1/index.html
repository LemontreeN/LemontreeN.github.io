如何实现一个简易http代理服务器

发表于2021-10-30|更新于2021-11-09|计算机网络
|字数总计:1.5k|阅读时长:7分钟|阅读量:

此为HIT计算机网络Lab1内容
1. 设计并实现一个基本 HTTP 代理服务器。要求在指定端口（例如 8080）接收来自客户的 HTTP 请求并且根据其中的 URL 地址访问该地址所指向的 HTTP 服务器（原服务器），接收 HTTP 服务器的响应报文，并将响应报文转发给对应的客户进行浏览。
2. 设计并实现一个支持 Cache 功能的 HTTP 代理服务器。要求能缓存原服务器响应的对象，并能够通过修改请求报文（添加 if-modified-since头行），向原服务器确认缓存对象是否是最新版本。（选作内容，加分项目，可以当堂完成或课下完成）
3. 扩展 HTTP 代理服务器，支持如下功能：（选作内容，加分项目，可以当堂完成或课下完成）
4. 网站过滤：允许/不允许访问某些网站；
5. 用户过滤：支持/不支持某些用户访问外部网站；
6. 网站引导：将用户对某个网站的访问引导至一个模拟网站（钓鱼）。

实验原理和过程

  1. 创建套接字，绑定好端口和ip地址，并持续监听，直到接收到请求开始服务
  2. 接收请求头，对其进行解析，得到请求方法，请求主机，请求端口
  3. 检测是否为屏蔽网站(发送403 Forbidden)、屏蔽主机(发送403 Forbidden)、钓鱼目标网址(发送回钓鱼内容)
  4. 建立套接字和目标主机通信
  5. 检测是否存在cache、cache是否为最新
  6. Cache最新，直接返回cache内容，无cache/cache需要更新，创建/更新cache，并发回给请求端
  7. 关闭除监听套接字以外的套接字

问题讨论

  1. 暂时不会实现https的cache功能
    访问https网站会先发送一个connect请求，回复“HTTP/1.X 200 Connection Established\r\n\r\n”后，由于https是加密通信，clinet_socket再发送过来的数据为加密数据，我暂时无法修改报文添加If Modified属性，应该是需要SSL知识
  2. 不知道为什么https代理加载速度非常快，而http代理加载速度很慢

心得体会

代理服务器本质功能是转发。
在http下，只需与目标建立连接，转发请求并且将获得的反馈发送给请求者即可；
而在https下，额外需要向请求方发送建立连接报文，再转发请求方的加密数据并且将获得的反馈发送给请求者。
对HTTP报文和运作方式了解加深了很多，能够改造报文实现某些功能。
